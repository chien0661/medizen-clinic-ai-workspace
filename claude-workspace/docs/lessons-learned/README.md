## How to Add a Lesson Learned

After fixing a difficult technical issue, use the `/publish-lesson-learn` skill:

```bash
/publish-lesson-learn
```

The skill will:
1. ✅ Guide you through creating a structured lesson learned entry
2. ✅ Create a local file in this directory: `LESSON-XXX.md`
3. ✅ Generate Confluence-formatted content
4. ✅ Provide instructions to publish to Confluence via MCP
5. ✅ Update this index automatically

## Categories

Lessons are categorized by technical area:

- **Database**: Query optimization, connection issues, migrations, data integrity
- **API**: REST/GraphQL issues, contract problems, integration failures
- **Performance**: Memory leaks, CPU spikes, slow queries, caching issues
- **Security**: Vulnerabilities, authentication, authorization, data exposure
- **Infrastructure**: Deployment issues, container problems, networking, resource limits
- **Integration**: Third-party service issues, MCP problems, external API failures
- **Frontend**: UI bugs, state management, rendering issues, browser compatibility
- **Backend**: Business logic bugs, error handling, service failures

---

## Severity Levels

- **Critical**: Production outage, data loss, security breach
- **High**: Significant functionality broken, major performance degradation
- **Medium**: Feature broken, moderate performance impact
- **Low**: Minor issue, workaround available
