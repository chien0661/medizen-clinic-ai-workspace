# BUG-066-001: Stale FE unit tests still test old MOCK_DATA fallback behavior

**Task:** TASK-066
**Date:** 2026-05-31
**Reporter:** Test Agent
**Type:** UI/E2E (FE unit test)
**Severity:** Medium

---

## Summary

Two unit tests in `src/tests/reports/ARAgingReportPage.test.tsx` were written against the old MOCK_DATA fallback behavior and now fail because the implementation correctly removed that fallback. The tests need to be updated to reflect the new error-state behavior.

## Failing Tests

| Test | File | Line |
|------|------|------|
| `renders with mock fallback data when BE unavailable` | `src/tests/reports/ARAgingReportPage.test.tsx` | 90 |
| `renders mock patient data in table` | `src/tests/reports/ARAgingReportPage.test.tsx` | 105 |

## Root Cause

`ARAgingReportPage.tsx` previously had:
```typescript
const MOCK_DATA: ARAgingResponse = { ... } // hardcoded 30.1M
// line 158-159:
} catch {
  return MOCK_DATA;  // silent fallback
}
```

TASK-066 correctly **removed** this fallback. When the API fails the component now shows an error state (`data-testid="ar-aging-error"`) instead of mock data.

The unit tests mock `api.get` to reject with `new Error("BE not available")` then assert that:
1. `"Chi tiết theo bệnh nhân"` heading is visible (the patient table — only shown when `data` is truthy)
2. `"Nguyễn Văn An"` patient name appears (old mock patient name)

Both assertions fail correctly because:
- When API rejects, `isError=true` and the error state is shown
- `data` is undefined so the patient table section is not rendered
- Neither "Chi tiết theo bệnh nhân" nor "Nguyễn Văn An" appear

## Expected Fix

Update the two failing tests to match the new behavior:

1. **`renders with mock fallback data when BE unavailable`** → rename to `shows error state when BE unavailable` and assert:
   - `data-testid="ar-aging-error"` is visible
   - Error message "Không thể tải dữ liệu báo cáo công nợ" is shown
   - Patient table section is NOT rendered

2. **`renders mock patient data in table`** → replace with a test that:
   - Mocks `api.get` to return a valid `ARAgingResponse` with real patient data
   - Asserts the patient name appears in the table when API succeeds
