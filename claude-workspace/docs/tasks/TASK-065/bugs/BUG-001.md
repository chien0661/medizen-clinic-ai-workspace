# BUG-001 — VssIntegrationConfigPage test uses api.patch but implementation uses api.put

**Task**: TASK-065
**Bug ID**: BUG-001
**Type**: UI/E2E (FE unit test mismatch)
**Severity**: Medium
**Found by**: Test Agent
**Date**: 2026-05-31

---

## Summary

The FE test `VssIntegrationConfigPage.test.tsx` was written when the save handler was a placeholder using `api.patch`. TASK-065 correctly updated the implementation to call `api.put("/api/v1/integrations/vss/config", config)`, but the test was NOT updated to match. As a result:

1. The `vi.mock("../../lib/apiClient")` only mocks `{ get, patch }` — `put` is not mocked.
2. The test `"calls api.patch on save form submit"` calls `expect(api.patch).toHaveBeenCalled()` — but the component now calls `api.put`, so `api.patch` is never called.
3. The test **fails with**: `AssertionError: expected "spy" to be called at least once`

---

## Type
UI/E2E (FE unit test)

## Severity
Medium — the implementation is correct; only the test is wrong. Actual save functionality works correctly. However the failing test breaks CI.

## SRS Reference
TASK-065 acceptance criteria: `GET/PUT /integrations/vss/config` must work.

## Detail Design Reference
`clinic-cms-web/src/pages/admin/VssIntegrationConfigPage.tsx` line 113:
```ts
await api.put("/api/v1/integrations/vss/config", config);
```

## Expected vs Actual Behavior

**Expected**: The test should mock `api.put` and assert `api.put` was called with the correct path and payload.

**Actual**: The test mocks `api.patch` and asserts `api.patch` was called — causing a failure because the component calls `api.put`.

## Fix Required

**File**: `clinic-cms-web/src/tests/admin/VssIntegrationConfigPage.test.tsx`

**Change 1** — Add `put` to the mock (line 26–29):
```ts
vi.mock("../../lib/apiClient", () => ({
  api: {
    get: vi.fn(),
    patch: vi.fn(),  // keep for other tests that may use patch
    put: vi.fn(),    // ADD — needed for save config
  },
}));
```

**Change 2** — Update `beforeEach` to also mock `put` (line 112):
```ts
(api.put as ReturnType<typeof vi.fn>).mockResolvedValue({ ok: true });
```

**Change 3** — Update the failing test (line 203–215):
```ts
it("calls api.put on save form submit", async () => {
  // ...
  fireEvent.click(screen.getByTestId("save-config-btn"));
  await waitFor(() => {
    expect(api.put).toHaveBeenCalledWith(
      "/api/v1/integrations/vss/config",
      expect.objectContaining({ api_url: expect.any(String) })
    );
  });
});
```
