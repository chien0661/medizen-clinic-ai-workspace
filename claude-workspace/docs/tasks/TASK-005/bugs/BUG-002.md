# BUG-002: Patient Create Accepts Future date_of_birth Without Validation

**Type:** Business Logic Bug
**Severity:** Medium
**Status:** OPEN
**Feature:** TASK-005 - Patient Management
**Found By:** Test Agent
**Date:** 2026-04-27

## SRS Reference
- Patient module requires valid demographic data; a future date_of_birth is logically impossible for a living patient.

## Detail Design Reference
- `app/modules/patients/schemas/patient_schemas.py` — `PatientCreateRequest` schema.
- The `date_of_birth` field has no future-date validator.

## Test Scenario
- **File:** `tests/integration/patients/test_patients_negative.py`
- **Scenario:** `TestCreateNegativePaths::test_create_future_dob_returns_4xx`
- Sends `POST /api/v1/patients` with `date_of_birth: "2099-01-01"`.

## Expected Behavior
- A `date_of_birth` in the future (e.g., `2099-01-01`) must be rejected with **400 Bad Request** or **422 Unprocessable Entity**.
- No patient record should be created.

## Actual Behavior
- The API accepts `date_of_birth: "2099-01-01"` and creates a patient with `birth_year=2099`.
- Returns **201 Created** — a logically invalid patient record is stored in the database.

## Steps to Reproduce
1. Authenticate as admin user.
2. `POST /api/v1/patients` with body:
   ```json
   {"full_name": "Future DOB Test", "gender": "female", "date_of_birth": "2099-01-01"}
   ```
3. Observe `201 Created` response with a patient record containing `birth_year=2099`.
4. Expected: `400`/`422` response.

## Test Output
```
FAILED tests/integration/patients/test_patients_negative.py::TestCreateNegativePaths::test_create_future_dob_returns_4xx
AssertionError: Future DOB should return 400/422, got 201: {"data": {"patient_code": "BN0001", ..., "birth_year": 2099, ...}}
assert 201 in (400, 422)
```

## Impact
- **Data Integrity:** Future birth years (e.g., 2099) create nonsensical patient records that may break age-based clinical rules.
- **Business Impact:** Downstream systems that rely on `birth_year` for age calculations (medication dosing, eligibility checks) may produce incorrect results.

## Suggested Fix
In `app/modules/patients/schemas/patient_schemas.py`, add a Pydantic validator on `date_of_birth`:
```python
from datetime import date
from pydantic import field_validator

@field_validator("date_of_birth")
@classmethod
def dob_must_not_be_future(cls, v: date | None) -> date | None:
    if v is not None and v > date.today():
        raise ValueError("date_of_birth cannot be in the future")
    return v
```
