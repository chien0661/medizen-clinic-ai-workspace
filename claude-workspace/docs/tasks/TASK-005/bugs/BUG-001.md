# BUG-001: Null Byte in Search Query Causes Unhandled 500 Error

**Type:** API Contract Issue
**Severity:** High
**Status:** OPEN
**Feature:** TASK-005 - Patient Management
**Found By:** Test Agent
**Date:** 2026-04-27

## SRS Reference
- Security / Input Validation: all user-supplied query strings must be sanitized before passing to the database layer.

## Detail Design Reference
- `app/modules/patients/services/patient_service.py` — `search_patients()` function.
- The search query `q` is passed directly to SQLAlchemy ORM expressions (both `plainto_tsquery` and trigram `%` operator) without null-byte stripping.

## Test Scenario
- **File:** `tests/integration/patients/test_patients_negative.py`
- **Scenario:** `TestSearchNegativePaths::test_search_null_byte_q_does_not_500`
- Sends `GET /api/v1/patients/search?q=test%00name&type=name`

## Expected Behavior
- Null bytes in query parameters must be sanitized (stripped or rejected) before reaching the database.
- The API must return **400 Bad Request** or **422 Unprocessable Entity** — never HTTP 5xx.

## Actual Behavior
- The null byte propagates to PostgreSQL via asyncpg, which raises:
  `asyncpg.exceptions.CharacterNotInRepertoireError: invalid byte sequence for encoding "UTF8": 0x00`
- FastAPI global exception handler converts this to a **500 Internal Server Error**.
- The error leaks an internal stack trace in server logs.

## Steps to Reproduce
1. Authenticate as any valid clinic user.
2. `GET /api/v1/patients/search?q=test%00name&type=name` (URL-encoded null byte `%00`).
3. Observe `500 Internal Server Error` response.
4. Expected: `400`/`422` response.

## Test Output
```
FAILED tests/integration/patients/test_patients_negative.py::TestSearchNegativePaths::test_search_null_byte_q_does_not_500
asyncpg.exceptions.CharacterNotInRepertoireError: invalid byte sequence for encoding "UTF8": 0x00
sqlalchemy.exc.DBAPIError: ... CharacterNotInRepertoireError: invalid byte sequence for encoding "UTF8": 0x00
AssertionError: Null byte in q should not cause 5xx, got 500: {"detail":"Internal server error"}
```

## Impact
- **Security Impact:** Null bytes can be used as injection vectors in some contexts; must be rejected at input.
- **Reliability Impact:** Unhandled 500 errors may crash the request handler and expose internal stack traces.
- **User Impact:** Any search with malformed input causes an unhandled error.

## Suggested Fix
In `app/modules/patients/api/routes.py` or `patient_service.search_patients()`:
1. Add a validator on the `q` query parameter that strips or rejects null bytes:
   ```python
   if '\x00' in q:
       raise HTTPException(status_code=400, detail="Query contains invalid characters")
   ```
2. Alternatively, apply a Pydantic validator on the `q` field in the route signature.
3. Consider a general-purpose query string sanitizer middleware or dependency.

## Related Files
- `app/modules/patients/api/routes.py` — search endpoint parameter validation
- `app/modules/patients/services/patient_service.py` — `search_patients()` function
- `app/modules/patients/schemas/patient_schemas.py` — consider adding a Pydantic field validator
