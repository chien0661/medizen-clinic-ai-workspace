# BUG-001: Create Visit with Non-Existent patient_id Raises Unhandled FK IntegrityError (Not 404)

**Task**: TASK-007
**Filed by**: Test Agent
**Date**: 2026-04-27
**Status**: OPEN

---

## Classification

- **Type**: Business Logic / Data Integrity
- **Severity**: MAJOR
- **SRS Reference**: Visit module — create walk-in visit
- **Detail Design Reference**: `app/modules/visits/services/visit_service.py::create_visit`

---

## Summary

When `POST /api/v1/visits` is called with a `patient_id` that does not exist in the database, the service does not pre-validate patient existence. It proceeds to insert the visit row, which triggers a PostgreSQL FK constraint violation (`fk_visit_patient_id_patient`). This `ForeignKeyViolationError` propagates as an unhandled `sqlalchemy.exc.IntegrityError` through the ASGI layer — resulting in an unhandled exception rather than a clean HTTP 404 response.

---

## Expected Behavior

`POST /api/v1/visits` with a non-existent `patient_id` should return:
```
HTTP 404 Not Found
{ "error": "Patient <uuid> not found" }
```

The service should SELECT the patient by `(id, clinic_id)` before inserting and raise `NotFoundError` if not found.

---

## Actual Behavior

The service calls `db.flush()` after adding the visit object, which triggers the FK constraint:

```
asyncpg.exceptions.ForeignKeyViolationError: insert or update on table "visit" violates
foreign key constraint "fk_visit_patient_id_patient"
DETAIL: Key (patient_id)=(<uuid>) is not present in table "patient".
```

This propagates as `sqlalchemy.exc.IntegrityError` and is not caught by the application's exception handlers, resulting in an unhandled Python exception (not HTTP 500 either — it propagates through the ASGI test transport).

---

## Steps to Reproduce

```bash
# With a valid admin token for clinic A:
POST /api/v1/visits
{
  "patient_id": "00000000-0000-0000-0000-000000000000",  # non-existent
  "chief_complaint": "test"
}
# Expected: 404 Not Found
# Actual: IntegrityError (FK violation) — not caught
```

---

## Root Cause

`visit_service.py::create_visit` does not validate patient existence before calling `db.flush()`. The patient check is missing:

```python
# Missing in create_visit():
patient = await db.get(Patient, patient_id)
if patient is None or patient.clinic_id != clinic_id:
    raise NotFoundError(f"Patient {patient_id} not found")
```

---

## Suggested Fix

In `app/modules/visits/services/visit_service.py`, add a patient existence check at the start of `create_visit()`:

```python
from app.modules.patients.models.patient import Patient  # adjust import

async def create_visit(...):
    # Validate patient belongs to the same clinic
    result = await db.execute(
        select(Patient).where(
            Patient.id == patient_id,
            Patient.clinic_id == clinic_id,
            Patient.is_deleted.is_(False),
        )
    )
    if result.scalar_one_or_none() is None:
        raise NotFoundError(f"Patient {patient_id} not found in clinic {clinic_id}")
    
    today = visit_date or date.today()
    ...
```

## Impact

- **User Experience**: The API returns an unhandled error rather than a descriptive 404.
- **Data Integrity**: No data corruption risk — the FK prevents the bad insert from committing.
- **Security**: No security risk — the patient doesn't exist so no data leaks.
- **Priority**: Medium — affects UX and API correctness but not data integrity.
